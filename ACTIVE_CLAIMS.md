# Active Claims

## Parallel Work Rules
- Pair this file with `MULTI_AGENT_PLAN.md` when the repo is actively using multiple agents.
- Claim substantial work here before editing when multiple agents are active.
- Prefer disjoint file ownership.
- Detailed claim records live under `methodology/claims/<claim-id>.md`.
- Every active claim should include a lease, a heartbeat timestamp, a based-on commit, and a takeover rule.
- Worker claim records should also capture exact files changed, commands run, result summary, known risks, integration notes, and merge/rebase readiness.
- Release or update a claim when the task or working set changes.
- If a claim expires, another agent may take it over only after recording the takeover in the claim file and `MULTI_AGENT_PLAN.md`.
