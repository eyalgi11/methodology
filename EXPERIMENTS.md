# Experiments

Use this file for active hypothesis-driven work. Keep it focused on experiments that are still proposed, running, or awaiting a keep/discard decision.

## Experiment Rules
- Do not promote an experiment into standard behavior until it beats the baseline and the result is recorded in `EXPERIMENT_LOG.md`.
- Every experiment should be bounded:
  - hypothesis
  - owner
  - primary metric
  - baseline
  - success threshold
  - time-to-signal
  - max budget / max runs
  - stop rule
  - rollback rule
- Preferred statuses:
  - `proposed`
  - `running`
  - `observing`
  - `winner`
  - `loser`
  - `inconclusive`
  - `promoted`
